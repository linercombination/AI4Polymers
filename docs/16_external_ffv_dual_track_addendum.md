# 外部 FFV 双轨预训练补充研究计划

这份文档是对主研究方案的补充，用来明确说明为什么外部 `FFV` 支线现在要同时做 `2D graph` 和 `3D graph` 两条预训练路线。

## 1. 变更原因

如果只训练一个外部 `FFV` 上游模型，后续即使下游任务得到提升，也很难判断收益来自：

- 图表示本身
- 空间坐标信息
- 或仅仅是训练波动

因此，外部 `FFV` 支线现在也采用和主任务一致的“可解释比较”策略。

## 2. 新的 FFV 研究结构

### 上游对比

1. `graph_2d -> FFV`
2. `graph_3d -> FFV`

### 下游对比

1. `baseline`
2. `baseline + predicted_ffv_2d`
3. `baseline + predicted_ffv_3d`
4. `oracle_ffv`

## 3. 预期输出

### 上游

- 2D 预训练模型指标
- 3D 预训练模型指标
- 两条路线的验证与测试集预测结果

### 下游

- 使用 `predicted_ffv_2d` 的主任务配置
- 使用 `predicted_ffv_3d` 的主任务配置
- 与 `baseline`、`oracle_ffv` 的统一对照表

## 4. 当前阶段结论口径

在没有完整下游对比结果之前，文档和实验应统一表述为：

- 当前已经具备 `2D` 与 `3D` 两条外部 `FFV` 预训练实现
- 后续将比较两者的上游预测能力与下游增益
- 不预设 `3D` 一定优于 `2D`
